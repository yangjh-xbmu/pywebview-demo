### 完整技术栈规划 (最终版)

#### **后端 (Backend)** - 稳定且安全

* **框架 (Framework)**: **FastAPI**
* **数据库 (Database)**: **SQLite**
* **数据库 ORM**: **SQLAlchemy** (配合 `aiosqlite` 驱动)
* **数据验证**: **Pydantic**
* **数据库迁移**: **Alembic**
* **用户与角色管理**: **`fastapi-users`**
  * *理由*: 提供了完整的用户认证（Authentication）生命周期管理。
* **权限管理 (Authorization)**: **自定义 FastAPI 依赖项**
  * *理由*: 基于 `fastapi-users` 获取到的用户及其角色，通过自定义依赖项实现灵活的基于角色的访问控制 (RBAC)。这是 FastAPI 推荐的模式，优雅且解耦。
* **Web 服务器**: **Uvicorn**

#### **前端 - 主应用 (Frontend - Main App)** - 灵活且美观

* **框架 (Framework)**: **Next.js**
* **UI Foundation**: **Tailwind CSS**
  * *理由*: 原子化 CSS 框架，提供了无与伦比的设计灵活性和控制力。
* **UI 组件库 (UI Component Library)**: **shadcn/ui**
  * *理由*: 这不是一个传统的组件库。它提供了一系列你可以直接复制粘贴到自己项目中的、设计精美的组件。这些组件基于 Radix UI（无头组件）和 Tailwind CSS 构建，意味着你可以**完全拥有并自由定制**每一个组件的代码，而不是安装一个黑盒 NPM 包。
* **状态管理**:  **React Context / Hooks**
* **数据请求**:  **React Query (TanStack Query)**
* **表单处理**: **React Hook Form** (与 `shadcn/ui` 组件完美集成)
* **动画库 (Optional)**: **Framer Motion**
  * *理由*: 为你的现代化 UI 添加流畅、精美的动画效果。

#### **前端 - 后台管理 (Frontend - Admin Panel)** - 高效且强大

* **核心框架**: **Refine**
* **元框架**: **Next.js**
* **UI 组件库**: **Ant Design (AntD)** & **Ant Design ProComponents**
  * *理由*: 提供企业级的、开箱即用的后台组件，如图表、高级表格、复杂表单等，极大加速后台界面的开发。
* **图表库**: **Ant Design Charts**

#### **桌面应用封装 (Desktop App Wrapper)**

* **核心库**: **PyWebView**
  * *理由*: 轻量级、跨平台，完美连接 Python 后端与 Web 前端。

#### **自动化测试 (Testing)**

* **后端测试 (Python/FastAPI)**:
  * **测试框架**: **pytest**
  * **HTTP 客户端**: **HTTPX** (通过 `FastAPI.TestClient`)
  * **代码覆盖率**: **pytest-cov**
* **前端测试 (JS/Next.js/React)**:
  * **测试框架**: **Jest**
  * **组件测试**: **React Testing Library (RTL)**
  * **端到端测试**: **Playwright**

#### **开发运维与代码质量 (DevOps & Code Quality)**

* **代码管理/Monorepo**: **Turborepo**
  * *理由*: 高效管理你的主应用和后台管理这两个前端项目。
* **代码格式化**: **Prettier** (用于 JS/TS/CSS) & **Black** (用于 Python)
* **代码质量检查 (Linting)**: **ESLint** (用于 JS/TS) & **Ruff** (用于 Python, 极速)
* **Git Hooks**: **Husky** & **lint-staged**
  * *理由*: 在你每次 `git commit` 时自动运行格式化和 lint 检查，确保提交到代码库的代码永远是干净、规范的。
* **持续集成 (CI)**: **GitHub Actions**
  * *理由*: 自动化运行测试、lint 检查，保障代码质量。

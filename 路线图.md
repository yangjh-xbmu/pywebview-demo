# 项目开发路线图 (分阶段任务)

我们将遵循“后端先行，纵向切片”的原则。先搭建好基础，然后选择一个功能（比如用户管理），从后端 API 到前端 UI 完整地实现它，而不是先把所有后端写完再写所有前端。

## 第一阶段：项目奠基 (Foundation Setup)

这个阶段的目标是搭建一个稳固的、自动化的开发环境。

### 任务 1: 初始化 Monorepo 和代码质量工具

目标: 创建一个统一的代码库，并配置好代码规范。

步骤:

1. 使用 npx create-turbo@latest 初始化 Turborepo 项目。
2. 使用 venv 初始化 Python 虚拟环境。
3. 在根目录配置配置 Ruff
4. 在根目录配置 ESLint (共享配置) 和 Prettier。
5. 在根目录配置 TypeScript 配置文件 (tsconfig.json)。
6. 在根目录配置 markdownlint
7. 在根目录配置 .gitignore 文件，忽略必要文件。
8. 在根目录配置 Husky 和 lint-staged，设置 pre-commit 钩子，在提交代码时自动格式化和检查代码python、typescript、markdown。

验收标准:

当你修改一个 `.ts`、`.py`、`.md` 文件并尝试 `git commit` 时，代码会自动被格式化和检查。

### 任务 2: 搭建基础 FastAPI 后端和数据库

目标: 运行一个能响应请求的 FastAPI 服务器，并连接到数据库。

步骤:

1. 在 monorepo 中创建一个 packages/backend (或类似) 目录来存放 FastAPI 应用。
2. 安装 FastAPI, Uvicorn, SQLAlchemy, aiosqlite。
3. 创建一个简单的 main.py，包含一个根路径 / 的 "Hello World" 端点。
4. 配置 SQLAlchemy 连接到 SQLite 数据库文件。
5. 集成 Alembic，定义第一个 User 模型，并生成并运行第一个数据库迁移。

验收标准:

1. 可以在浏览器或 API 工具中访问 http://127.0.0.1:8000 看到 "Hello World"，并且数据库文件中已创建了用户表。

## 第二阶段：核心认证与授权 (Core Auth)

这个阶段的目标是实现完整的用户登录注册和权限控制。

任务 3: 集成 fastapi-users

目标: 实现用户注册、登录、登出等核心认证功能。
步骤:
按照 fastapi-users 的文档，将其与你的 SQLAlchemy User 模型和 FastAPI 应用集成。
启用 JWT 认证策略。
fastapi-users 会自动生成 /register, /login, /logout 等路由。
验收标准: 使用 Postman 或 Insomnia 等 API 工具，可以成功调用 /register 注册新用户，然后用新用户的信息调用 /login 获取到 JWT token。
任务 4: 实现基于角色的权限管理 (RBAC)

目标: 创建一个只有管理员才能访问的 API 端点。
步骤:
修改 User 模型和 Alembic 迁移，增加一个 role 字段 (例如，字符串类型，值为 'user' 或 'admin')。
创建一个 FastAPI 依赖项 (Dependency)，例如 def get_admin_user(...)，它会验证当前登录的用户是否为管理员。如果不是，则抛出 HTTPException。
创建一个新的测试端点，例如 /admin/dashboard，并使用你刚刚创建的依赖项来保护它。
验收标准:
使用普通用户的 token 访问 /admin/dashboard 返回 403 Forbidden。
手动将数据库中某个用户的角色改为 'admin'，使用该用户的 token 访问 /admin/dashboard 成功。
第三阶段：前端搭建与连接 (Frontend MVP)
现在后端核心已就绪，我们可以开始构建看得到的东西了。

任务 5: 初始化前端应用并实现登录页面

目标: 创建两个前端应用的骨架，并能通过 UI 调用后端的登录接口。
步骤:
在 Turborepo 的 apps/ 目录下，分别创建两个 Next.js 应用：main-app 和 admin-app。
在 main-app 中：
使用 npx shadcn-ui@latest init 初始化 shadcn/ui。
安装 TanStack Query 和 React Hook Form。
创建一个登录页面，使用 shadcn/ui 的 Input, Button, Form 组件。
使用 useMutation (来自 TanStack Query) 和 React Hook Form 来处理表单提交，调用后端的 /login API。
登录成功后，将 JWT token 存起来（例如在 Context 或 cookie 中），并跳转到首页。
在 admin-app 中：重复上述登录页面的逻辑，但使用 Ant Design 的组件。
验收标准: 可以在主应用和后台应用的登录页面输入用户名密码，成功登录并跳转。
任务 6: 实现后台管理的用户列表

目标: 在后台管理系统中，使用 Refine 快速生成一个完整的用户管理 CRUD 界面。这是一个完美的“纵向切片”。
步骤:
后端: 创建 /users 的 CRUD API 端点，并用管理员权限进行保护。
前端 (admin-app):
集成 Refine 及其 Ant Design 适配器。
定义一个 "users" 资源 (resource)，并将其 list, create, edit 属性指向你刚刚创建的后端 API。
Refine 会自动使用 Ant Design 的 Table, Form 等组件为你生成功能齐全的用户管理页面。
验收标准: 以管理员身份登录后台，可以看到用户列表，并可以对用户进行编辑和创建。
第四阶段：功能扩展与完善 (Feature Expansion)
基础已经非常牢固，现在可以并行业务功能的开发。

任务 7: 开发主应用核心功能 (例如：文章发布系统)

目标: 在主应用中实现一个完整的业务模块。
步骤:
后端: 设计并实现文章 (Article) 相关的模型、数据库迁移和 CRUD API。API 需要权限保护（例如，只有登录用户才能创建文章）。
前端 (main-app):
创建文章列表页、详情页、新建/编辑页。
大量使用 shadcn/ui 构建美观的界面。
使用 useQuery 获取文章列表/详情，使用 useMutation 处理创建/更新操作。
任务 8: 桌面应用打包

目标: 将你的 Web 应用封装成一个桌面应用。
步骤:
编写一个 run_desktop.py 脚本。
该脚本首先在一个后台线程中启动 FastAPI/Uvicorn 服务器。
然后，使用 PyWebView 创建一个窗口，其 URL 指向你的前端开发服务器 (例如 http://localhost:3000) 或构建后的静态文件。
验收标准: 运行 python run_desktop.py 后，会弹出一个桌面窗口，里面加载了你的主应用，并且所有功能（如登录、看文章）都正常工作。
第五阶段：测试与部署 (Polish & Deploy)
任务 9: 编写自动化测试

后端: 使用 pytest 和 HTTPX 为关键的 API 逻辑（特别是权限部分）编写单元测试和集成测试。
前端: 使用 React Testing Library 为关键组件（如登录表单）编写单元测试。使用 Playwright 编写一个关键的端到端 (E2E) 测试用例，例如 "用户登录 -> 创建文章 -> 退出登录"。
任务 10: 配置持续集成 (CI)

目标: 自动化执行测试和代码检查。
步骤:
在项目根目录创建 .github/workflows/ci.yml 文件。
配置 GitHub Actions，使其在每次 push 或 pull request 时：
安装 Python 和 Node.js 环境。
运行后端的 ruff 和 pytest。
运行前端的 eslint 和 jest/playwright 测试。
验收标准: 当你向 GitHub 推送代码时，Actions 会自动运行，并报告所有检查和测试是否通过。
